# Environmental Consulting Patterns

## 1. Research and Demonstration Report

Common purpose: support project decision, approval communication, technical route selection, or management feasibility judgement.

Recommended logic:

> background need -> key question -> basis and boundary -> current condition -> technical route or scenario -> impact/risk comparison -> feasibility judgement -> conditions and follow-up work

Emphasize what the report can support and what still needs design, monitoring, approval, or special review.

## 2. Environmental Impact Analysis

Recommended logic:

> impact source -> receptor -> pathway -> indicator -> baseline -> prediction/comparison -> impact judgement -> mitigation and monitoring

For water, air, noise, soil, groundwater, ecology, solid waste, and environmental risk sections, state the impact source and receptor explicitly. Do not only write "response characteristics" or "impact situation".

## 3. Pollution Control or Remediation Study

Recommended logic:

> problem identification -> source/pathway analysis -> control target -> alternative comparison -> preferred measure -> implementation condition -> performance verification

When comparing measures, use the same criteria such as removal efficiency, risk reduction, construction difficulty, operation stability, cost level, secondary impact, and maintenance need.

## 4. Environmental Risk Assessment

Recommended logic:

> risk source -> scenario -> trigger condition -> affected object -> consequence range -> existing prevention capacity -> residual risk -> emergency and monitoring requirements

Use cautious conclusions. Prefer "risk has control conditions after implementation of..." over "there is no risk".

## 5. Monitoring Plan

Recommended logic:

> monitoring purpose -> object/location -> indicator -> frequency -> period -> method/standard -> trigger condition -> data use

A monitoring recommendation should explain how results will support operation adjustment, warning, acceptance, follow-up assessment, or regulatory reporting.

## 6. Expert Review Response

Recommended logic:

> comment -> treatment decision -> revised basis -> revised content -> location in report -> remaining boundary

Do not answer only with "adopted" or "revised". Make the actual modification traceable.

## 7. Government-Facing Technical Briefing

Recommended logic:

> project issue -> technical finding -> risk boundary -> management condition -> requested coordination or decision point

Use concise language. Avoid dense methodological explanation unless it directly supports the decision.
